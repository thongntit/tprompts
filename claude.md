# tprompts

Universal CLI for installing prompt packs directly into your coding editors.

## Overview

`tprompts` is a command-line tool that helps you install AI prompts into any code editor with a simple one-line command. It allows you to:
- **Install prompts directly from GitHub URLs** - no registration required
- **Support any editor name** - not limited to predefined editors (vscode, cursor, windsurf, claude, roo, etc.)
- **Preview changes before installing** with `--dry-run`
- **Remove installed prompts** when no longer needed

## Installation

```bash
npm install -g @thongntit/tprompts
```

Or use with npx:
```bash
npx @thongntit/tprompts <command>
```

## Quick Start

### Direct Installation (Recommended)

The simplest way to use tprompts - install directly from a GitHub URL:

```bash
# Install a prompt from GitHub
tprompts install https://github.com/thongntit/prompts/tree/main/prompts/obisidian-second-brain claude

# Works with any editor name your prompt pack supports
tprompts install https://github.com/user/repo/prompt-name vscode
tprompts install https://github.com/user/repo/prompt-name cursor
tprompts install https://github.com/user/repo/prompt-name roo

# Preview before installing
tprompts install https://github.com/user/repo/prompt-name claude --dry-run

# Force overwrite existing files
tprompts install https://github.com/user/repo/prompt-name vscode --force
```

## Available Commands

### `install` - Install a prompt

Install a prompt pack from a GitHub URL into your current project for a specific editor.

```bash
# Install from GitHub URL
tprompts install https://github.com/user/repo/prompt-name vscode
tprompts install https://github.com/user/repo/tree/main/prompt-name cursor
tprompts install https://github.com/thongntit/prompts/tree/main/prompts/obisidian-second-brain claude

# Preview changes without installing
tprompts install https://github.com/user/repo/prompt-name vscode --dry-run

# Force overwrite without confirmation
tprompts install https://github.com/user/repo/prompt-name cursor --force
```

**Options:**
- `-e, --editor <editor>` - Target editor (any name supported by the prompt pack)
- `-f, --force` - Overwrite existing files without confirmation
- `--dry-run` - Preview what would be installed without making changes

## Prompt Repository Format

Each prompt pack must include a `tprompts.json` configuration file:

```json
{
  "name": "Coding Assistant",
  "description": "AI coding assistant with best practices",
  "version": "1.0.0",
  "editors": {
    "vscode": {
      "system-prompt.md": {
        "location": ".vscode/ai-assistant.md",
        "prefix": "---\nrole: system\napplyTo: '**'\n---\n\n"
      },
      "commands/": {
        "location": ".vscode/commands/",
        "suffix": "\n\n<!-- Generated by tprompts -->"
      }
    },
    "cursor": {
      "system-prompt.md": {
        "location": ".cursorrules"
      }
    },
    "windsurf": {
      "system-prompt.md": {
        "location": ".windsurfrules"
      }
    },
    "claude": {
      "system-prompt.md": {
        "location": ".claude/custom-instructions.md"
      },
      "commands/": {
        "location": ".claude/commands/"
      }
    }
  }
}
```

**Configuration options:**
- `location` - Destination path relative to project root
- `prefix` - Text to add before file content (optional)
- `suffix` - Text to add after file content (optional)
- Directory support - Use trailing `/` to copy entire directories recursively

## Supported Editors

Any editor name can be used! Common examples include:
- **VS Code** - `.vscode/` directory
- **Cursor** - `.cursorrules` file and `.cursor/` directory
- **Windsurf** - `.windsurfrules` file and `.windsurf/` directory
- **Claude Code** - `.claude/` directory
- **Roo** - Custom configuration
- Any other editor - Just add it to your `tprompts.json`

## Tips

- Use `--dry-run` to preview changes before applying them
- The CLI validates that the requested editor is supported by the prompt pack
- Temporary files are automatically cleaned up after installation
