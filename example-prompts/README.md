# Example Prompts Repository

This is an example repository demonstrating the `tprompts.json` configuration format for the universal tprompts CLI.

## Repository Structure

```
example-prompts/
├── .tprompts-repo.json          # Repository metadata (optional)
├── README.md                    # This file
├── coding-assistant/            # Individual prompt project
│   ├── tprompts.json           # Configuration for this prompt
│   ├── system-prompt.md        # Main system prompt
│   └── coding-rules.md         # Additional coding guidelines
└── obsidian-workflow/          # Another prompt project
    ├── tprompts.json           # Configuration for this prompt  
    ├── general-system.md       # Main system prompt
    └── commands/               # Directory of command files
        ├── weekly-review.md
        └── daily-planning.md
```

## Configuration Features Demonstrated

### Editor-Specific Configurations
Each prompt supports different editors with custom paths and processing:

- **VSCode**: Files go to `.vscode/` with frontmatter prefixes
- **Cursor**: Files go to `.cursorrules` and `.cursor/` directories
- **Windsurf**: Files go to `.windsurf/` with custom naming
- **Claude Code**: Files go to project root with descriptive names

### Advanced File Processing

#### Prefix/Suffix Support
```json
{
  "system-prompt.md": {
    "location": ".vscode/ai-assistant.md",
    "prefix": "---\napplyTo: '**'\nrole: 'system'\n---\n\n",
    "suffix": "\n\n<!-- Generated by tprompts -->"
  }
}
```

#### Directory Handling
```json
{
  "commands/": {
    "location": ".vscode/obsidian-commands/",
    "prefix": "// Obsidian Command\n// Generated by tprompts\n\n"
  }
}
```

## Usage Examples

### Register this repository
```bash
# Register as local repository
tprompts register /path/to/example-prompts --name "examples"

# Or clone and register (if this was a git repo)
tprompts register https://github.com/user/example-prompts.git
```

### Install prompts
```bash
# Install coding assistant to VSCode
tprompts install examples/coding-assistant vscode

# Install obsidian workflow to Cursor  
tprompts install examples/obsidian-workflow cursor

# List available prompts
tprompts list examples
```

### Results

After installing `coding-assistant` to `vscode`, you would get:
```
.vscode/ai-assistant.md        # system-prompt.md with VSCode frontmatter
.vscode/coding-standards.md    # coding-rules.md with prefix
```

After installing `obsidian-workflow` to `windsurf`, you would get:
```
.windsurf/obsidian-rules.md           # general-system.md with prefix
.windsurf/workflows/obsidian/weekly-review.md    # with suffix
.windsurf/workflows/obsidian/daily-planning.md   # with suffix
```

## Creating Your Own Prompts

1. Create a directory for your prompt project
2. Add a `tprompts.json` configuration file
3. Add your prompt files (markdown recommended)
4. Configure editor-specific paths and processing
5. Test with `tprompts install --dry-run`

See the `tprompts.json` files in this repository for configuration examples.
